# ログアウト機能 設計

## 目的

Cloudflare Access 経由で Google 認証した利用者が、家計簿アプリの設定画面から明示的にログアウトできるようにする。

## スコープ

- 設定画面の末尾にログアウト導線を追加する。
- 誤操作を防ぐ確認ダイアログを表示する。
- Cloudflare Access の利用者セッションを終了する。
- Google アカウント自体のセッションは終了しない。

ログイン機能、許可メールアドレスの管理、セッション期限、Google アカウント全体からのログアウトは変更対象外とする。

## 採用方式

確認後、ブラウザを同一オリジンの `/cdn-cgi/access/logout` に遷移させる。

Cloudflare Access が公式に提供するログアウトエンドポイントを直接利用するため、独自の API ルートや環境変数は追加しない。アプリケーションドメイン上のエンドポイントを使うことで、そのドメインの認証 Cookie が即座に削除される。

Cloudflare Access は利用者自身によるアプリ単位のログアウトを提供していない。この操作では同じ Access 組織のほかの保護対象アプリを含む Access セッションが失効する一方、認証元である Google アカウントのセッションは維持される。

## UI と操作フロー

1. `SettingsPage` の既存設定項目より後に「アカウント」カードを表示する。
2. カードには、このアプリからログアウトする操作であることを示す短い説明と、danger スタイルの「ログアウト」ボタンを配置する。
3. ボタンを押すと、既存の `ConfirmDialog` で「ログアウトしますか？」と確認する。
4. 「キャンセル」またはダイアログを閉じた場合は、ダイアログだけを閉じる。
5. 「ログアウト」を確定した場合は `window.location.assign("/cdn-cgi/access/logout")` を実行する。
6. Cloudflare Access がセッションを終了し、以後の保護対象ページへのアクセスでは再認証を要求する。

## コンポーネント境界

- `SettingsPage` が確認ダイアログの開閉状態とログアウト遷移を管理する。
- 表示には既存の `Card`、`Button`、`ConfirmDialog` を再利用する。
- サーバー側の Hono API と Access JWT 検証ミドルウェアは変更しない。

今回の処理は単一画面内の単純な遷移であり、専用フックや認証サービスは新設しない。

## エラーと環境別の扱い

ログアウト処理は Cloudflare 管理エンドポイントへのページ遷移であり、クライアント側で待機状態や API エラーを処理しない。Cloudflare Access が有効な本番環境で機能する。

ローカル開発では `DEV_BYPASS_ACCESS=true` により認証自体を省略しているため、実際のセッション終了は検証対象外とする。ローカルでボタンを確定すると同じパスへ遷移するが、Cloudflare Access のログアウト応答は得られない。

## 検証

- `npm test` で既存の API・ドメインロジックに退行がないことを確認する。
- `npm run build` で TypeScript と本番ビルドが成功することを確認する。
- 本番相当の Cloudflare Access 配下で、次の手動確認を行う。
  - 設定画面末尾にログアウトカードが表示される。
  - キャンセルで設定画面に留まる。
  - 確定で Cloudflare Access のログアウト処理へ遷移する。
  - ログアウト後に保護対象ページへ戻ると再認証を求められる。

## 完了条件

- 利用者が設定画面から確認付きでログアウトできる。
- キャンセル時に認証状態が変化しない。
- ログアウト時に Google アカウント全体のセッションを終了しない。
- 既存テストと本番ビルドが成功する。
